WHAT YOU CAN DO WITH THIS SKILL
================================================================================

A. Build a new microservice from scratch:
   1. Read quick-start.md for overview
   2. Use fastapi-patterns.md as boilerplate
   3. Add business logic
   4. Build Docker image using Dockerfile.fastapi
   5. Test with docker-compose.yml

B. Deploy to Kubernetes:
   1. Run: python scripts/bootstrap-k8s.py --service myservice --image myrepo/myservice:v1
   2. Review generated manifests
   3. Apply: kubectl apply -f k8s/
   4. Monitor with Kubernetes dashboard

C. Setup production observability:
   1. Run: python scripts/setup-monitoring.py --services service1 service2
   2. Deploy: kubectl apply -f monitoring/
   3. Access Prometheus, Grafana, Jaeger UIs
   4. Configure alerts based on SLOs

D. Implement advanced patterns:
   - Service mesh: Use Istio manifests from advanced-patterns.md
   - Events: Implement Kafka producers/consumers
   - Security: Apply mTLS, network policies
   - Resilience: Add circuit breakers, bulkheads

E. Debug production issues:
   - Use distributed traces to find slow requests
   - Query Prometheus for metrics
   - Search logs with correlation IDs
   - Check health check endpoints

PRODUCTION CHECKLIST (from SKILL.md)
================================================================================

Before deploying to production, verify:

Architecture:
  - Service boundaries clear (bounded contexts)
  - Each service owns its data
  - Async communication for loose coupling
  - API versioning strategy defined

Implementation:
  - Structured logging with correlation IDs
  - Metrics and health checks on all services
  - Graceful shutdown handlers
  - Configuration via environment variables
  - Secret management (not in code/config)

Deployment:
  - Container images build and run correctly
  - K8s manifests or docker-compose validated
  - Resource requests/limits configured
  - Health checks (liveness/readiness probes)
  - Auto-scaling policies defined

Observability:
  - Structured logs to centralized system
  - Key metrics exposed and alerting configured
  - Distributed tracing end-to-end
  - Dashboards for on-call visibility

Security:
  - Service-to-service authentication (mTLS or JWT)
  - Data encrypted in transit and at rest
  - Network policies restricting traffic
  - Secret rotation automated
  - RBAC configured per environment

Testing:
  - Unit tests (business logic)
  - Integration tests (against dependencies)
  - Contract tests (API contracts between services)
  - Load/chaos tests (resilience validation)
